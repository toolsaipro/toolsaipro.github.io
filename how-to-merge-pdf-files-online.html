How the Merging Works — The Technology Behind It

Understanding what actually happens during the merge process confirms why ToolsAI Pro is genuinely private. Here is the exact technical pipeline:

  1. FileReader API: Each selected PDF is read into a Uint8Array (binary array) stored in your browser's RAM. This is a local memory operation with zero network activity.
  2. pdf-lib Loading: The pdf-lib library parses each PDF's internal structure — its pages, fonts, embedded images, vector graphics and metadata — entirely within your browser tab.
  3. Page Copying: pdf-lib's copyPages() function copies each page from the source PDFs into a new blank PDF document in the exact order you specified. No re-rendering occurs — the page content is copied at the byte level.
  4. Assembly & Save: The new merged PDF is assembled with pdfDoc.save() using object streams for optimal file size. The output exists only in browser memory.
  5. Download: A Blob URL is created from the merged bytes and a programmatic download is triggered. The file goes directly from browser memory to your device's download folder — no server involved at any stage.

You can verify this yourself: open Chrome DevTools (F12 → Network tab), then upload and merge your files. You will see zero outbound network requests during the entire process. This is the proof that your files never leave your device.
